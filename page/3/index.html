学习日志20161107

发表于 2016-11-07 | 分类于 日志

https://gank.io/post/560e15be2dca930e00da1083

在仔细地看这篇文章，对监听器模式有了基本的概念，响应式编程方面看懂了许多。

创建Observable时会自动地将读入的一整个对象拆成分散的对象，在OnSubscribe的call方法中定义订阅时进行的数据操作和对subscriber的回调。再进行subscriber中onNext与onCompleted的定义来实现读取流中的对象并做出相应的动作。

将所有的操作、数据等抽象为一个个流，因为流是线性的，对流的过滤、变换、合并等操作就可以一步步线性地执行， 以这样的形式来完成特定的功能。

后面开始讲异步的内容：这一点应该是最方便的地方，通过给subscriber与observe分别设置不同的线程预设值，非常简洁地就把取数据的操作和显示数据的操作放在了不同的线程中，可以实现常见的后台取数据、主线程显示的策略。也可以想象这样简便的分离方法可以在异步操作上发挥很大的作用。

我收回前面说的最方便的话……下面讲到的map变换和flatmap变换真是可怕 。。

map可以理解成简单地将一个流经过一些针对对象的处理转化为另一个类型的对象，而flatmap的返回则是一个新的observable对象，这个对象的初始化可以由前一个observable中的一些iteratable对象完成，等于说把原来流中的对象又展开成了一个新的流，并且整合在一起重新发送。再由subcriber接收对象进行处理。这样的机制使得灵活性可以很大，但代码上的逻辑仍是一个个对流的处理，逻辑上简洁易懂。

今天就看到这里。

一周总结20161106

发表于 2016-11-06 | 分类于 日志

在这周的第一天搞定了github的学生优惠，拿到了digitalocean的五十刀，重新搭了ss，装了锐速以后竟然可以看油管的4k。。装lnmp，放了个wordpress搭的博客上去，搞定了之前没有弄的ftp，用之前的域名(viseator.xyz)完成了个人博客。

一天弄vps的过程竟然比想象中的顺利不少，与之前折腾的两周的linux是分不开的。也就是在同一天ubuntu上的androidStudio再次挂掉，换回windows。

当天晚上接到了第一周的新人任务， 查了一些metrial design的内容以后就直接开始做了，这一周几乎所有时间都花在了这个APP上。到最后也算是将基本任务做完了。

做这个APP的感受就是没有一个十分清晰的思路，写代码都是想到哪里写到哪里，并且没有加任何的注释，更没有考虑一些方法的封装问题，在后面事实上写的一部分代码和前面写过的是高度相似的。虽然可以正常跑起来，但是我觉得过一段时间自己都会看不懂了。

不过之前也从没想过一个礼拜的时间真的可以从零撸出一个APP，从无到有，功能一点一点完善的过程还是充满了乐趣的。

 

虽然APP完成了，但问题还是很多的。

其中的一些代码是从网上的示例代码照抄过来的，没有进行太深入的了解，但是有一些关键的地方经过几天的不断修改完善也能明白其中的一些原理了（例如Adapter)。

java语言本身的基础不够，没有对语言进行深入的学习的后果就是面向对象的思想没有建立 ，对一些语法上的内容并不熟悉，在写功能时更多的还是面向过程的。在之后需要加强对java的学习。

对编译器的操作不熟练， 一些重要的快捷键不知道或是不太会用，断点调试还是不会用， 查看文档等功能也是刚知道。

 

晚上的分享讲的是retrofit和rxjava相关的内容，retrofit有一些http基础可以听懂一些，对于rxjava的部分不是太理解，但是观察者模式的概念可以理解一些了，分享的那篇长博客讲得非常详细，明天会仔细阅读，在今后的项目中应该会用上。

 

在任务代码检查的时候mentor给了许多宝贵的建议，例如命名、格式、分包等等的规范问题，也提到由于第一个任务的代码量较少这些方面的不注意不会有太大的问题，但在今后的项目过程中这些规范非常地重要，在以后我也会认真注意这些问题。

各外mentor给我推荐了一些优秀的网站和资源，尽量在英语不是太好的情况下看起来比较痛苦，但我想我会坚持看的。

 

总结下来，正如之前招新的时候说的，来联创，一个礼拜学java，一个礼拜写出第一个自己的app……这样的体验……

 

接下来应该会迎来第二个任务，我想在做这个任务的时候我会像组长说的，慢下来仔细地想一些问题，尽量弄清楚一些更深入的东西，并且锻炼自己面向对象的思维，同时对java语言本身继续进行学习。如果有空闲时间的话leetcode的算法题也应该会开始做。

还是希望自己可以在新人期期间快速成长，争取早日进入可以接锅的状态。

看了设置监听器的示例代码，发现如果要为不同的部件设置不同的监听器就应该设置不用的ViewHolder从而将部件分离开。

发现，ViewHolder的分离是针对于不同的Item的，但我这是一个Item中有不同的部件，故只需在onBindViewHolder中设置相关的监听就可以了，并不是要监听针对Item的点击事件。成功为Checkbox设置点击事件并将值存入数据库中。

关于滑动删除：

http://blog.csdn.net/u010687392/article/details/47950199
在写滑动删除代码时遇到问题：之前查找数据库中的数据都是根据position+1对应到数据库中设置的自增的id，然后找到数据，一旦删除一条数据后，id中就会空出一个，在生成的时候就找不到对应的数据了。

目前想到的解决办法是设置一个方法用于更新数据库中的id值， 并且在新增时id不能再为自增，需要进行手动设置。

利用了count方法手动设置新加入数据的id，在删除一条数据后通过循环将之后的数据的id全部前移一位。完成了滑动删除功能。
