学习日志20161118

2016-11-19

实现用AsyncTask在后台请求数据，并储存在String中。

发现AsyncTask对线程的调度比用Tread的那一套高到不知道哪里去了，没有踩很多的坑，花了一些时间来理解几个过程执行的线程和顺序，在doInBackground读取数据以后传到onPostExecute中进行Gson解析，并将解析后获得的自定义类的List传入Adapter中进行view的加载。

Gson这次看的是官方文档，虽然是英文的但是讲得还是很清楚的，终于明白了熬测时候一头包的Type到底是个什么东西了，在定义解析出的类时还是踩了一些坑，官方文档给出了一个示例来说明错误的情况：

public class A {
  public String a;

class B { 

public String b; 

public B() {
  // No args constructor for B
}

}
}
然后说

NOTE: The above class B can not (by default) be serialized with Gson.

Gson can not deserialize {"b":"abc"} into an instance of B since the class B is an inner class. If it was defined as static class B then Gson would have been able to deserialize the string. Another solution is to write a custom instance creator for B.
并没有给出static的实现，自己在类中定义了一个静态内部类以后并不能正确解析嵌套的json，而且怎么获得内部类的域也让我纠结了很久，开始以为是访问级别设置的问题，但是改来改去还是不能正确解析，最后google了很久发现可以把类定义在外面，然后再写方法去取里面的数据，这样以后就可以正确解析了。

通过这个过程我发现自己对内部类和访问级别这些内容不是太清楚，写代码的时候很迷茫，不能去正确分析原因，准备在之后再去复习java这方面的内容。

解析正确以后就是一些简单的view设置了。

开始做根据url读取头像并显示的操作。
